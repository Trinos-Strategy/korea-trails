# Korea Trails — 데스크톱 임팩트 + Gemini 생성 자산 업그레이드 프롬프트

> **대상 에이전트** — Antigravity(Gemini) 에이전트. `feat/rebranding-overhaul` 브랜치에서 **이어서** 작업.
> **이번 트랙의 두 축** — ① **데스크톱(웹브라우저) 최적화를 별도 1순위 페이즈로 분리** ② **Gemini 이미지·영상 생성**으로 아이콘·이미지·영상을 만들어 삽입·배치.
> **문제의식** — 현재 결과는 (a) 모바일 중심이라 데스크톱 임팩트가 약하고, (b) Lighthouse 100/100은 기술 점수일 뿐 **아트디렉션 임팩트와 무관**하다. 이 트랙은 "큰 화면에서 압도적인가"를 1차 목표로 한다.

---

## 0. 운영 원칙 (에이전트 필독)

- **기술 점수 ≠ 임팩트**: 접근성/SEO 100점을 유지하되, **별도의 "데스크톱 임팩트 합격 기준"(§3)** 을 통과해야 한다. 점수만으로 완료 선언 금지.
- **데스크톱 퍼스트, 모바일 무손상**: 이번엔 **큰 화면(≥1280, 와이드 ≥1536)을 먼저 설계**하고 작은 화면으로 우아하게 축소한다. 단, 기존 모바일 동작을 깨지 않는다.
- **같은 브랜치 유지**: `feat/rebranding-overhaul`에서 이어가고, 페이즈/페이지 단위 커밋. main 직접 푸시 금지. PR을 실제로 연다.
- **빌드리스·다크/라이트·a11y·무회귀·정보손실0** 하드제약 유지.
- **두 정지점 준수**: P1(데스크톱 히어로 시안) 사용자 승인 전 전체 롤아웃 금지, P3(자산 아트디렉션 1산 파일럿) 승인 전 26산 생성 금지.

## 1. PHASE A — 데스크톱/웹브라우저 최적화 (먼저, 단독)

자산 생성보다 **먼저** 레이아웃·타이포·임팩트를 데스크톱 기준으로 끌어올린다. (자산 없이도 골격이 웅장해야 한다.)

### A1. 레이아웃 시스템
- 콘텐츠 컨테이너 `max-width: clamp(1080px, 90vw, 1440px)`, 와이드(≥1536)에서 좌우 여백·그리드 확장.
- 랜딩 카드 그리드: 데스크톱 **3~4열**, 와이드 4~5열, `gap` 확대. 모바일에서 1~2열로 축소.
- 플레이북: 데스크톱에서 **2단 레이아웃**(예: 좌측 코스 본문 + 우측 sticky 지도/요약), 모바일에서 1단 스택.
- 헤더: 데스크톱 가로 내비(코스 앵커·테마·검색), 모바일 햄버거/컴팩트.

### A2. 임팩트 타이포 & 히어로
- **시네마틱 히어로**: 최소 높이 `min(88vh, 920px)`, 풀블리드, 대형 디스플레이 헤드라인(데스크톱 `clamp(3rem, 6vw, 6.5rem)`), 강한 위계.
- 디스플레이/본문 폰트 페어링 재점검, 행간·자간·측정폭(measure) 데스크톱 최적화.
- 깊이감: 레이어드 그라디언트 오버레이, 미세 그레인/비네팅, 스크롤 시 히어로 패럴랙스(약하게, reduced-motion 분기).

### A3. 데스크톱 인터랙션
- 카드 hover lift/zoom, 포커스 비주얼, 필터/검색의 데스크톱 친화 배치(상단 바 + 결과 카운트).
- sticky 코스 내비, 스크롤 진행 표시, 섹션 reveal 모션.

### A4. PHASE A 게이트
- 데스크톱(1280/1440/1920/2560)·태블릿·모바일 스크린샷으로 **반응 검증**(chrome-devtools MCP 활용).
- §3 임팩트 기준 자기점검 + **데스크톱 히어로 시안 사용자 승인(P1 정지점)**.

## 2. PHASE B — Gemini 생성 자산 파이프라인

Gemini의 이미지(Imagen/Nano Banana류)·영상(Veo류) 생성을 적극 활용해 자산을 만들고, 최적화해 삽입한다.

### B1. 아이콘 세트 (이모지 전면 대체)
- Gemini로 **일관된 아이콘 시스템** 시드 생성(라인 두께·코너·메타포 통일): 산·고도·거리·시간·난이도·버스/교통·온천·텐트/산장·단풍·설경·경고·팁 등.
- 생성물 → **SVG로 정리·최적화**(단색·currentColor, 다크/라이트 양용), `assets/icons/`에 심볼 스프라이트화. 래스터 PNG 직접 삽입 지양(아이콘은 벡터).
- 산출물: `assets/icons/*.svg` + `_assets/icon-spec.md`(메타포·사용처).

### B2. 이미지 (히어로·분위기·텍스처)
- **아트디렉션 통일**: 시간대·광질·색온도·구도·후처리를 한 문장 스타일 가이드로 고정해 26산이 한 시리즈로 보이게.
- **용도 구분**:
  - 히어로/대형: 산의 분위기·능선·계절감.
  - 보조: 텍스처(바위·안개·숲), 섹션 배경, OG/소셜 카드, 파비콘.
- ⚠️ **지리적 진위 가드레일**: AI 생성 이미지는 "그 산의 실제 지형"이 아닐 수 있다. **권장 하이브리드** — 산을 식별하는 핵심 히어로는 실제 사진(스톡)을, 분위기·텍스처·배경·아이콘·OG는 Gemini 생성으로. 전적 AI 생성 시 캡션에 **"AI 생성 이미지" 고지**.
- **출력 규격**: 가로형 고해상 → 리사이즈 [640/1024/1600/2400], AVIF+WebP+JPG, lazy(히어로 제외), `width/height`로 CLS 방지.
- 산별 생성 프롬프트 템플릿(B5).

### B3. 영상 (데스크톱 한정, 신중히)
- **랜딩 히어로 앰비언트 루프**(Veo류): 6~10초, 무음, `loop muted playsinline`, **poster 이미지 필수**, 자동재생은 데스크톱·고속회선만.
- **모바일·reduced-motion·저속회선 폴백**: 정적 히어로 이미지로 대체(영상 비로드).
- **성능 예산**: 히어로 영상 ≤ 3~4MB(압축·짧게), `preload="none"` + 뷰포트 진입 시 로드. 페이지 LCP는 poster 기준.
- 선택적: 산별 짧은 분위기 클립은 갤러리 내 클릭 재생(자동재생 금지)으로 제한.
- 산출물: `assets/video/<id>/hero.mp4`(+ webm), `hero-poster.jpg`.

### B4. 배치·삽입 규칙
- 디자인 시스템(`design-system.css`)의 히어로/갤러리/카드/캡션 컴포넌트에 자산을 주입(하드코드 인라인 지양).
- `_assets/asset-manifest.json`: 자산별 용도·경로·출처(Gemini/stock)·AI고지·alt(한/영)·attribution.
- `CREDITS.md`에 자산 출처·라이선스·AI 생성 고지 기록.

### B5. 산별 이미지 생성 프롬프트 템플릿 (예: 양명산)
```
[STYLE GUIDE 고정] cinematic outdoor photography, golden-hour soft light,
high dynamic range but natural, atmospheric depth, no text, no people in foreground,
16:9, premium travel-magazine grade.
[SUBJECT] Yangmingshan / Qixingshan, Taiwan — volcanic terrain, sulfur fumaroles,
silver-grass grassland, ridge trail, Taipei skyline faint in distance.
[USE] hero background (desktop full-bleed).
[NEGATIVE] cartoon, watermark, distorted geology, fake landmarks.
```
→ 26산 각각 SUBJECT만 교체. 진위 민감 산은 stock 히어로 + Gemini 보조로.

### B6. PHASE B 게이트
- **1산 파일럿(양명산 권장)** 으로 아이콘+히어로 이미지+(가능 시)영상까지 적용한 완성본을 만들고 **사용자 승인(P3 정지점)** 후 26산 확장.

## 3. 데스크톱 임팩트 합격 기준 (점수와 별개)

- [ ] 1920/2560 와이드에서 히어로가 **풀블리드·시네마틱**(여백 낭비·좁은 중앙단 정렬 아님)
- [ ] 카드 그리드가 데스크톱 3~4열로 가로 공간 활용, 와이드에서 확장
- [ ] 플레이북 데스크톱 2단 레이아웃(본문 + sticky 보조)
- [ ] 대형 디스플레이 타이포 위계가 첫 화면에서 강한 인상
- [ ] 이모지 0개 — 전부 일관 SVG 아이콘
- [ ] 히어로에 Gemini/stock 이미지(또는 영상) 적용, 분위기·깊이감 확보
- [ ] 영상은 데스크톱만 자동재생, poster·모바일·reduced-motion 폴백 동작, ≤4MB
- [ ] 모바일 무손상, 다크/라이트 정상, LCP<2.5s·CLS<0.1, A11y/SEO 100 유지
- [ ] 자산 manifest·CREDITS·AI 고지 완비, 정보손실0, 콘솔 에러 0

PR 실제 생성). 현재 결과는 두 가지 문제가 있다: (1) 모바일 중심이라 데스크톱 임팩트가 약하고,
(2) Lighthouse 100점은 기술 점수일 뿐 아트디렉션 임팩트가 부족하다. 아래 순서로 재작업하라.
근거 문서: DESKTOP-IMPACT-UPGRADE.md (단일 진실원천). 하드제약: 빌드리스·다크/라이트·a11y·
무회귀·정보손실0.

[PHASE A — 데스크톱 최적화 먼저, 자산 없이]
- 데스크톱(≥1280, 와이드 ≥1536)을 먼저 설계하고 모바일로 축소한다(모바일 무손상).
- 컨테이너 max-width clamp(1080px,90vw,1440px), 랜딩 카드 3~4열, 플레이북 데스크톱 2단(본문+sticky 보조),
  데스크톱 가로 내비.
- 시네마틱 히어로(min 88vh), 대형 디스플레이 타이포(clamp 3rem~6.5rem), 레이어드 오버레이·약한 패럴랙스
  (reduced-motion 분기), 카드 hover.
- chrome-devtools MCP로 1280/1440/1920/2560 스크린샷 검증.
- ★여기서 멈춰 데스크톱 히어로 시안을 사용자에게 보여주고 승인받는다(P1).

[PHASE B — Gemini 생성 자산]
- 아이콘: Gemini로 일관 아이콘 세트 생성 → SVG 최적화(currentColor, 다크/라이트) → 이모지 전면 대체.

- 영상(데스크톱 한정): 랜딩 히어로 앰비언트 루프 6~10초, muted/loop/playsinline, poster 필수,
  preload=none+뷰포트 진입 시 로드, ≤4MB, 모바일·reduced-motion은 정적 폴백.

먼저 PHASE A의 실행계획과 데스크톱 히어로 방향을 요약 제시한 뒤 승인받고 시작하라.
