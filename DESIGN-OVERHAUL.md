# Korea Trails — UI/UX/랜딩 전면 리디자인(리브랜딩) 오케스트레이션 설계서

> **모드** — **대담한 리디자인(리브랜딩)**. 새 비주얼 아이덴티티·타이포·레이아웃·컴포넌트 시스템을 재설계한다. 단, 기술 하드제약(빌드리스·GitHub Pages·다크모드·a11y·성능·무회귀)은 유지.
> **런타임** — Claude Code 서브에이전트. **산출물** — 프롬프팅 설계만(실행 별도).
> **마스터 설계서**(`ORCHESTRATION-UPGRADE.md`)·**샘플**(`SAMPLE-yangmingshan-upgrade.md`)의 보조·상위 문서. 이 트랙이 **새 디자인 시스템을 확정**하면 사진/지도 트랙이 그 위에 얹힌다.

## 1. 목표 & 디자인 노스스타

**미션**: "텍스트 카드 아카이브를 — 한국·대만 명산을 탐험하고 싶게 만드는, 사진·지도 중심의 프리미엄 아웃도어 프로덕트로 리브랜딩한다."

**디자인 노스스타 3원칙**
1. **신뢰(Trust)** — 산행 정보 제품. 정확·가독·계층이 미감보다 우선.
2. **욕망(Desire)** — 첫 스크롤에서 "가고 싶다". 실사·여백·타이포 위계로 감정 유발.
3. **일관(System)** — 23개 페이지가 하나의 디자인 시스템으로 묶인다. 산마다 다른 마크업을 통합.

**측정 가능한 성공**

| 지표 | 현재 | 목표 |
|---|---|---|
| 디자인 시스템 일원화 | 페이지별 분기(2종+ 템플릿) | 공유 `design-system.css` 1소스 |
| 랜딩 핵심 기능 | 정적 그리드만 | 검색·지역/난이도 필터·정렬·(선택)전국 지도 |
| 비주얼 위계 | 이모지 아이콘·텍스트 카드 | 아이콘 시스템·사진 카드·명료한 타이포 스케일 |
| 첫인상(LCP/CLS) | 미측정 | LCP<2.5s, CLS<0.1 |
| a11y | 미측정 | WCAG 2.1 AA, Lighthouse A11y≥95 |
| SEO/공유 | 메타 빈약 | title/description/OG/twitter, sitemap |

## 2. 현황 진단 (디자인 감사 결과)

- **README ↔ 실제 불일치(기능 부채)**: README는 "산 이름 검색·지역별 필터·인터랙티브 전국 지도"를 표방하나 **`index.html`엔 없음**(정적 그리드 렌더만). → 리디자인에서 **약속된 기능을 실제 구현**.
- **템플릿 분기**: seoraksan류(`--color-*`, `course-*` 탭)와 yangmingshan류(`--surface2`, `tabpane`)가 공존. CSS·클래스·테마 토글(`[data-theme-toggle]` vs `#themeBtn`)이 제각각 → **디자인 부채**.
- **아이코노그래피**: 🏔🥾⏱♨ 이모지 의존 → 플랫폼별 렌더 편차·비프리미엄. → **SVG 아이콘 세트**로 교체.
- **랜딩 빈약**: 단색 그라디언트 히어로(사진 없음), 텍스트 카드, 단일 섹션, 기본 푸터. 정렬·필터·빈상태·로딩·검색 없음.
- **강점(계승)**: OKLCH 토큰 사고, 라이트/다크, clamp 타이포, 반응형 그리드 — **시스템적 토대는 양호**하므로 리브랜딩의 출발점으로 재활용.

## 3. 제약 — 리브랜딩 모드 (마스터 §3 일부 완화)

**완화(이 트랙 한정)**
- 색·타이포·토큰·레이아웃 **재정의 허용**(새 아이덴티티). 단 새 토큰은 라이트+다크 양쪽 정의 필수.
- **공유 `assets/css/design-system.css` + `assets/js/app.js` 도입 허용** — 파일 링크는 "빌드 도구"가 아니며 GitHub Pages 정적 호스팅과 호환. 23개 페이지의 중복 인라인 CSS를 공유 시트로 수렴(점진적).

**불변(유지)**
- 빌드리스(번들러·npm 런타임 금지), GitHub Pages에서 파일 그대로 동작.
- 다크/라이트·반응형·키보드 접근·`prefers-reduced-motion`·`prefers-color-scheme`.
- 기존 콘텐츠(코스·체크포인트·고도·팁)의 **정보 손실 0**. 기능(탭·아코디언) 무회귀.
- 변경은 브랜치+PR, 페이지 단위 커밋. main 직접 푸시 금지.
- **방향 승인 게이트 통과 전 대량 롤아웃 금지**(§5).

## 5. 단계 DAG & 게이트

| Phase | 내용 | 담당 | 게이트(통과조건) |
|---|---|---|---|
| **P0 감사** | 현황·부채·기회 인벤토리 | DA-AUDIT | `design-audit.md` + `debt-inventory.json` |
| **P1 방향확정** | 2~3 방향안 제시 → **사용자 선택** | DA-DIRECTION | ★ **사용자 승인 1개 방향**(필수 정지점) |
| **P2 시스템** | 토큰·타이포·아이콘·`design-system.css`·컴포넌트 규격 | DA-SYSTEM | `design-tokens.json` + `design-system.css` + `components.md` |
| **P3 파일럿** | 랜딩 + 플레이북 1개에 시스템 적용 | DA-LANDING·DA-PLAYBK (+MOTION/A11Y/CONTENT) | **파일럿 2페이지 QA GREEN** |

## 7. 랜딩(index) 리디자인 상세 — DA-LANDING

### 7.1 구조(위→아래)
1. **히어로** — 풀블리드 명산 사진(사진 트랙 연계) + 그라디언트 오버레이 + 헤드라인·서브카피·1차 CTA(탐색 시작)·검색바. 통계(등록 26·플레이북 16·난이도 3단계)는 보조 위치로.
2. **탐색 컨트롤(신규)** — **검색**(산 이름 한/영/한자), **필터**(지역·난이도·국가·완성여부), **정렬**(고도·난이도·이름). URL 쿼리 동기화(공유 가능), `app.js`로 구현(무프레임워크).
3. **(선택) 전국 지도** — Leaflet 한국+대만 마커 클러스터, 클릭 시 카드/플레이북 이동. README 약속 이행. lazy.
4. **카드 그리드(재설계)** — 사진 썸네일 + 난이도/지역 뱃지 + 핵심 스탯(고도·거리·시간) + 상태. 호버·포커스 모션. 빈상태/검색결과없음 처리.
5. **섹션 분류** — 한국 / 해외(대만) 구획, 추후 테마(단풍·설경·당일·종주) 콜렉션 확장 여지.
6. **푸터** — 브랜드·출처·라이선스(CREDITS)·예약 링크·소셜/OG.

### 7.2 기술
- `app.js`: 클라이언트 검색/필터/정렬(현 `MOUNTAINS[]` 확장: `image`,`country`,`tags`). 디바운스 검색, `aria-live` 결과 카운트, 키보드 내비.
- SEO/메타: `<title>`/`description`/canonical/OG/twitter, `mountains` 데이터로 JSON-LD(`TouristAttraction`) 고려, `sitemap.xml`.
- 성능: 히어로 이미지 preload, 카드 썸네일 lazy+`width/height`, 폰트 `font-display:swap`.

## 8. 플레이북 UI 시스템 통합 — DA-PLAYBK

- **컴포넌트 규격 단일화**: 히어로, 코스 셀렉터/탭, 구간 카드, 아코디언, 체크포인트 테이블, 고도 프로파일, 난이도 바, 팁/경고 박스, 뱃지, 지도 컨테이너, 크레딧 캡션, 교통 섹션(샘플 연계).
- **마이그레이션 전략**: 각 페이지를 `design-system.css`에 링크하고 클래스 매핑(`--surface2`→토큰, `#themeBtn`→공유 테마 컴포넌트). `debt-inventory.json` 기준 페이지별 처리.
- **콘텐츠 보존 검증**: 마이그레이션 전후 텍스트·수치·링크 diff로 정보 손실 0 확인. `[web:xxx]` 잔재 제거(사실 불변).
- **파일럿 우선**: yangmingshan(분기 템플릿) + seoraksan(표준 템플릿) 2개로 양극단 검증 후 확장.

## 9. 모션 · 접근성 · 성능 (DA-MOTION / DA-A11Y-PERF / DA-CONTENT)

- **모션**: 카드 호버 lift, 필터 전환, 스크롤 reveal, 테마 전환 — 전부 `prefers-reduced-motion`에서 비활성/축소. 과한 패럴랙스 금지(산행 정보 신뢰 우선).
- **접근성**: 대비 AA, 포커스 링 가시성, 키보드 전동선, 스킵 링크, 의미적 랜드마크(`header/nav/main/footer`), 필터 `aria`, 지도 동등 정보(카드 그리드) 유지.
- **성능 예산**: 랜딩 above-the-fold ≤ 500KB, `design-system.css`/`app.js` 경량, 지도/이미지 lazy, LCP<2.5s·CLS<0.1·INP 양호.
- **콘텐츠/IA**: 라벨·마이크로카피·빈상태/에러/로딩 문구 한·영 일관, 난이도/지역 용어 표준화.

산출물: _orchestration/direction-options.md (시각 설명 + 토큰 초안). 가능 시 정적 HTML 미리보기 1장.
완료기준: 사용자가 1개 방향을 명시 승인. 승인 전 다음 Phase 진행 금지.
```

### DA-SYSTEM
```
역할: 디자인 시스템 엔지니어.
미션: 승인된 방향을 단일 소스 디자인 시스템으로 구현.
작업: design-tokens.json(라이트/다크 쌍) → assets/css/design-system.css(리셋+토큰+컴포넌트 유틸) → SVG 아이콘 세트(이모지 대체) → components.md(컴포넌트별 HTML 골격+사용예+a11y 노트).
완료기준: DA-LANDING/PLAYBK가 추가 결정 없이 구현 가능한 구체성, 하드코드 색 0, 다크/라이트 완비.
```

### DA-LANDING
```
역할: 랜딩 리디자인 구현자.
미션: index.html을 §7 구조로 재설계하고 약속된 기능(검색·필터·정렬·선택적 전국지도)을 구현.
작업: 히어로(사진+검색바)·탐색컨트롤(app.js, URL동기화·aria-live)·카드그리드(사진·뱃지·스탯·빈상태)·SEO/OG/JSON-LD·성능(preload/lazy).
산출물: index.html + assets/js/app.js (브랜치 커밋).
완료기준: 검색/필터/정렬 키보드 동작, 결과없음/로딩 처리, LCP<2.5s·CLS<0.1, 다크/반응형 정상.
```

### DA-PLAYBK
```

작업: design-system.css 링크 + 클래스 매핑, 컴포넌트 교체(탭/아코디언/테이블/뱃지/박스), 이모지→SVG아이콘, [web:xxx] 제거.
파일럿: yangmingshan + seoraksan 2개 먼저.
완료기준: 정보 손실 0(전후 diff), 기능 무회귀, 일관 컴포넌트, 콘솔 에러 0.
```

### DA-QA (독립 적대적)
```
1. 일관성: 23p가 동일 디자인 시스템·토큰·컴포넌트를 쓰는가(스타일 드리프트 탐지).
2. a11y: 대비 AA, 키보드 전동선, 랜드마크/aria, reduced-motion.
